┌──────────────────────────────────────────────────────────────────────────┐
│                     ResearchOrchestrator                                 │
│                                                                          │
│                        StateGraph Workflow                               │
│                                                                          │

│  │                 │      • Initialize state                            │
│  │  current_step:  │      • Add system message                          │
│  │  INITIALIZATION │      • Log query & focus areas                     │
│  └────────┬────────┘                                                    │
│           │                                                              │
│           ▼                                                              │
│  ┌─────────────────┐                                                    │
│  │    Research     │  ──> Execute web research                          │
│  │                 │      • Call web_research tool                      │
│  │  current_step:  │      • Parse JSON results                          │
│  │  RESEARCH       │      • Store raw_research_data                     │
│  └────────┬────────┘      • Track errors                               │
│           │                                                              │
│           ▼                                                              │
│  ┌─────────────────┐                                                    │
│  │   Synthesize    │  ──> LLM analysis of findings                      │
│  │                 │      • Create synthesis prompt                     │
│  │  current_step:  │      • Invoke LLM                                  │
│  │  SYNTHESIS      │      • Parse structured output                     │
│  └────────┬────────┘      • Store synthesized_findings                 │
│           │                                                              │
│           ▼                                                              │
│  ┌─────────────────┐                                                    │
│  │    Finalize     │  ──> Create final output                           │
│  │                 │      • Merge raw & synthesized data                │
│  │  current_step:  │      • Add timestamp                               │
│  │  FINALIZATION   │      • Structure final_output                      │
│  └────────┬────────┘      • Remove empty fields                        │
│           │                                                              │
│           ▼                                                              │
│      ┌────────┐                                                         │
│      │  END   │                                                          │
│      └────────┘                                                         │
│                                                                          │
│  • Explicit workflow control                                            │
│  • Customizable nodes and edges                                         │
│  • Typed state management                                               │
│  • LangChain v1.0 compatible                                            │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘


STATE MANAGEMENT COMPARISON
================================================================================

LEGACY (AgentExecutor):
─────────────────────────────────
    {
      "input": "query string",
      "output": "string result",
      // Internal state hidden
    }

MODERN (StateGraph):
─────────────────────────────────
    ResearchState:
      messages: List[BaseMessage]         # Message history
      query: str                          # Research question
      focus_areas: List[str]              # Focus topics
      raw_research_data: Dict             # Tool results
      synthesized_findings: Dict          # LLM analysis
      final_output: Dict                  # Structured result
      current_step: str                   # Workflow position
      errors: List[str]                   # Error tracking


KEY BENEFITS OF MIGRATION
================================================================================

1. EXPLICIT CONTROL
   Legacy: Fixed ReAct loop
   Modern: Custom nodes with specific purposes

2. STATE VISIBILITY
   Legacy: Hidden internal state
   Modern: Fully typed and trackable state

3. ERROR HANDLING
   Legacy: Try-catch around entire execution
   Modern: Error tracking per node

4. EXTENSIBILITY
   Legacy: Hard to modify workflow
   Modern: Easy to add nodes, edges, conditions

5. DEBUGGING
   Legacy: Black box execution
   Modern: Clear step-by-step logging

6. MULTI-AGENT INTEGRATION
   Legacy: Isolated execution
   Modern: Easy orchestration with other agents

7. FUTURE-PROOF
   Legacy: Deprecated in LangChain v1.0
   Modern: Recommended pattern going forward


EXAMPLE: Adding a Validation Node
================================================================================

Just add the node and update edges:

    ┌─────────────┐
    │  Synthesize │
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐  NEW NODE
    │  Validate   │  ──> Check synthesis quality
    └──────┬──────┘      Rate confidence level
           │              Flag low-quality results
           ▼
    ┌─────────────┐
    │  Finalize   │
    └─────────────┘

No need to modify the core loop logic!
