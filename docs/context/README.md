# Judge Dredd Project Context

Dette projekt følger **vibe-coding-context-structure** rammeværket for systematisk dokumentation.

## Struktur

### `/business/`
Forretningsdomæne og value proposition:
- `domain.md` - Business problem, target users, value proposition, success metrics

### `/product/`
Produktspecifikationer og features:
- `features.md` - Core features, API endpoints, roadmap

### `/engineering/`
Teknisk dokumentation:
- `architecture.md` - Tech stack, system design, data flow, security

## Opdatering
Dokumentationen skal opdateres når:
- Nye features tilføjes
- Arkitektur ændres
- Business requirements ændres
- Nye compliance frameworks tilføjes

## Brug
Før udvikling:
1. Læs relevant kontekst fra denne mappe
2. Følg workflows fra `/docs/commands/` (hvis relevant)
3. Opdater dokumentation efter større ændringer
